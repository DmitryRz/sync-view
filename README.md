**Sync View** — это full-stack веб-приложение, разработанное для совместного просмотра медиа-контента в реальном времени. Платформа позволяет пользователям создавать комнаты, делиться видео и синхронизировать воспроизведение, создавая эффект присутствия, даже находясь на расстоянии.

---

### 📜 О проекте

Этот проект является дипломной работой на тему: **"Проектирование и разработка платформы для совместного просмотра медиа-контента и обмена данными в реальном времени"**.

Основная идея — создать удобный сервис, где друзья, коллеги или члены семьи могут вместе смотреть фильмы, сериалы, лекции или любой другой видео-контент с идеальной синхронизацией и возможностью общения в чате.

---

### ✨ Ключевые возможности

*   **Аутентификация и авторизация**: Используется Keycloak для безопасного управления пользователями и их доступом.
*   **Загрузка видео**: Пользователи могут загружать свои видеофайлы на платформу, которые хранятся в MinIO.
*   **Создание комнат**: Пользователи могут создавать приватные или публичные комнаты для совместного просмотра.
*   **Синхронизация видео**: Воспроизведение, пауза и перемотка видео синхронизируются для всех участников комнаты в реальном времени.
*   **Чат в реальном времени**: Возможность обмена текстовыми сообщениями между участниками комнаты через WebSockets.
*   **Список пользователей**: Просмотр списка активных пользователей в системе.

### 🏗️ Архитектура приложения

Приложение использует микросервисную архитектуру, оркестрируемую с помощью `docker-compose`.

1.  **`gateway` (Nginx)**: Входящая точка приложения. Работает как reverse proxy, распределяя трафик:
    *   `/auth/` -> `keycloak`
    *   `/api/` -> `backend`
    *   `/storage/` -> `minio`
    *   Все остальные запросы -> `frontend`
2.  **`frontend` (React + Nginx)**: Клиентское приложение, разработанное с помощью React и Vite. После сборки статические файлы раздаются через отдельный Nginx-сервер, который упакован вместе с фронтенд-приложением в один Docker-образ.
3.  **`backend` (Spring Boot)**: Серверная часть, реализующая всю бизнес-логику, RESTful API и управление WebSocket-соединениями для синхронизации в реальном времени. Взаимодействует с PostgreSQL и MinIO.
4.  **`db` (PostgreSQL)**: Основная база данных для хранения информации о пользователях, комнатах, видео и сообщениях.
5.  **`keycloak`**: Сервер аутентификации, управляющий пользователями, ролями и сессиями. Интегрирован с собственной PostgreSQL базой данных (`db-keycloak`).
6.  **`minio`**: Локальное S3-совместимое объектное хранилище, используемое для хранения загруженных пользователями видеофайлов и аватаров.

Все сервисы изолированы и запускаются в Docker-контейнерах, что обеспечивает консистентность окружения и простоту развертывания.

### ⚙️ Переменные окружения (`.env.example`)

Для настройки проекта необходимо создать файл `.env` в корневой директории, скопировав `.env.example`. Важные переменные:

*   **`DB_USER`, `DB_PASSWORD`, `DB_NAME`**: Учетные данные для основной базы данных PostgreSQL.
*   **`DOMAIN_NAME`**: Доменное имя, используемое для доступа к приложению (например, `bintry.ru` или `localhost` для локальной разработки).
*   **`PROTOCOL`**: Протокол доступа (обычно `https://` или `http://`).
*   **`VITE_KEYCLOAK_URL`, `VITE_KEYCLOAK_CLIENT_ID`, `VITE_KEYCLOAK_REALM`**: Настройки для Keycloak на стороне фронтенда.
*   **`KEYCLOAK_ADMIN`, `KEYCLOAK_ADMIN_PASSWORD`**: Учетные данные администратора для Keycloak.
*   **`MINIO_ROOT_USER`, `MINIO_ROOT_PASSWORD`**: Учетные данные для доступа к консоли MinIO.
*   **`MINIO_BUCKET_NAME`**: Имя бакета для хранения файлов в MinIO.
*   **`BACKEND_ACCESS_KEY`, `BACKEND_SECRET_KEY`**: Ключи доступа для бэкенда для взаимодействия с MinIO.

### 🚀 Как запустить проект локально

Для запуска проекта вам понадобятся установленные **Docker** и **Docker Compose**.

#### 1. Настройка окружения

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/DmitryRz/sync-view
    cd sync-view
    ```

2.  **Создайте файл `.env`** в корневой директории проекта, скопировав `.env.example`:
    ```bash
    # Для Windows (PowerShell)
    copy .env.example .env

    # Для macOS / Linux
    cp .env.example .env
    ```
    Вы можете изменить значения в `.env` (например, пароли), но для первого запуска можно оставить стандартные.

    **Важно:** Для локальной разработки без реального домена, установите `DOMAIN_NAME=localhost` и `PROTOCOL=http://` в `.env`. Затем добавьте следующую строку в ваш файл `hosts` (обычно `/etc/hosts` на Linux/macOS или `C:\Windows\System32\drivers\etc\hosts` на Windows):
    ```
    127.0.0.1       localhost
    ```

#### 2. Запуск с помощью Docker Compose

1.  Откройте терминал в корневой директории проекта.
2.  Выполните команду для сборки и запуска всех контейнеров в фоновом режиме:
    ```bash
    docker-compose up -d --build
    ```
3.  Проект будет доступен в браузере по адресу:
    [http://localhost](http://localhost) (или `${PROTOCOL}${DOMAIN_NAME}` если вы изменили `DOMAIN_NAME`)

    **Дополнительно:**
    *   **Keycloak Admin Console**: [http://localhost:8180/auth/admin](http://localhost:8180/auth/admin) (используйте `KEYCLOAK_ADMIN`/`KEYCLOAK_ADMIN_PASSWORD` из `.env`)
    *   **MinIO Console**: [http://localhost:9001](http://localhost:9001) (используйте `MINIO_ROOT_USER`/`MINIO_ROOT_PASSWORD` из `.env`)

#### 3. Остановка проекта

Чтобы остановить все запущенные контейнеры, выполните команду:
```bash
docker-compose down
```

*   **`client/test-client.html`**: Простой HTML-файл для тестирования WebSocket-соединений и взаимодействия с чатом и плеером комнат. Откройте его в браузере, вставьте JWT-токен (полученный после аутентификации через Keycloak) и используйте для отправки сообщений или сигналов управления видео. Это полезно для отладки WebSocket-функциональности без использования полного UI фронтенда.
