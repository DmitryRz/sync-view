**Sync View** — это full-stack веб-приложение, разработанное для совместного просмотра медиа-контента в реальном времени. Платформа позволяет пользователям создавать комнаты, делиться видео и синхронизировать воспроизведение, создавая эффект присутствия, даже находясь на расстоянии.

---

### 📜 О проекте

Этот проект является дипломной работой на тему: **"Проектирование и разработка платформы для совместного просмотра медиа-контента и обмена данными в реальном времени"**.

Основная идея — создать удобный сервис, где друзья, коллеги или члены семьи могут вместе смотреть фильмы, сериалы, лекции или любой другой видео-контент с идеальной синхронизацией и возможностью общения в чате.

### 🏗️ Архитектура приложения

Приложение использует микросервисную архитектуру, оркестрируемую с помощью `docker-compose`.

1.  **`gateway` (Nginx)**: Входящая точка приложения. Reverse proxy, который распределяет трафик:
    *   Запросы, начинающиеся с `/api/`, перенаправляются на `backend`.
    *   Все остальные запросы — на `frontend`.
2.  **`frontend` (React + Nginx)**: Клиентское приложение, созданное с помощью Vite. После сборки статические файлы раздаются через собственный Nginx-сервер.
3.  **`backend` (Spring Boot)**: Серверная часть, реализующая всю бизнес-логику и API. Взаимодействует с базой данных и управляет WebSocket-соединениями для синхронизации в реальном времени.
4.  **`db` (PostgreSQL)**: Система управления базами данных для хранения информации о пользователях, комнатах и медиа.

Все сервисы изолированы и запускаются в Docker-контейнерах, что обеспечивает консистентность окружения и простоту развертывания.

### ✨ Ключевые возможности (в разработке)

*   **Создание комнат**: Пользователи могут создавать приватные или публичные комнаты для просмотра.
*   **Синхронизация видео**: Воспроизведение, пауза и перемотка видео синхронизируются для всех участников комнаты.
*   **Обмен данными в реальном времени**: Использование WebSockets для мгновенной передачи состояний и сообщений.
*   **Чат**: Возможность общения участников в комнате.

### 🚀 Как запустить проект локально

Для запуска проекта вам понадобится **Docker** и **Docker Compose**.

#### 1. Настройка окружения

Перед запуском необходимо настроить переменные окружения.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/DmitryRz/sync-view
    cd sync-view
    ```

2.  **Создайте файл `.env`** в корневой директории проекта, скопировав `.env.example`:
    ```bash
    # Для Windows (PowerShell)
    copy .env.example .env

    # Для macOS / Linux
    cp .env.example .env
    ```
    Вы можете изменить значения в `.env` (например, пароль для базы данных), но для первого запуска можно оставить стандартные.

#### 2. Запуск с помощью Docker Compose

1.  Откройте терминал в корневой директории проекта.
2.  Выполните команду для сборки и запуска всех контейнеров в фоновом режиме:
    ```bash
    docker-compose up -d --build
    ```
3.  Проект будет доступен в браузере по адресу:
    [http://bintry.localhost](http://bintry.localhost)

#### 3. Остановка проекта

Чтобы остановить все запущенные контейнеры, выполните команду:
```bash
docker-compose down
```
